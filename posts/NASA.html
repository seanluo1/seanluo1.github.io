NASA Internship Retrospective

August 12, 2018

Chicago, Illinois

I was extremely fortunate enough to work for NASA at The Johns Hopkins University Applied Physics Laboratory for the past 10 weeks. Situated in Laurel, Maryland, JHU APL was my first software engineering internship, so I was eager to see what it was like to be working on software outside of a classroom setting.

I worked on a team with 4 other software developers, a Scrum master, and product owner. We developed software according to the Agile (Scrum) guidelines. This was probably the best thing I learned from my experience this summer. Disecting a project into product backlog items, then further decomposing those items into subtasks was something that I've never done in the past, but I found to be an extremely useful skill. Frequent stand-up, PBI grooming, and Sprint planning/review meetings made sure that everyone was constantly working toward the same overarching goal, yet didn't step on each others toes while working on smaller tasks.

I had the role of a full-time software engineer. I handled my own subtasks, and had as much work to do as the rest of my coworkers, which was awesome. I worked on test software that created mission-independent test cases for NASA AMPCS, the ground system that communicates with almost all of NASA JPL's space vehicles. I wrote CLTU parsers in Python to read the data that was transmitted to the space craft, as well as a program to write these data packets in Java.

I gained experience in Git, Atlassian products (Bitbucket, Confluence, and Jira), and Agile methodologies.
